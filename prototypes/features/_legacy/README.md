# _legacy

## Purpose

`prototypes/features/_legacy/` is the temporary archive for older feature material that is being phased out of the new structure. It is not the active source of truth.

## Contract

- `prototypes/features/_legacy/` is archive-only and is not the active source of truth.
- Use it only for migrated historical feature material.
- New work belongs in `prototypes/features/<feature-slug>/`.
- Do not put shared masters or delivery pages here.

## What goes here

- Old feature pages during migration.
- Historical artifacts that still need a place while we clean up the tree.
- Transition-only content that is no longer the active source of truth.

## What should not go here

- New feature work.
- Shared masters.
- Anything that should be treated as a current feature directory.

## Relationship to `docs/specs` and `docs/plans`

Legacy content may point back to old `docs/specs/` or `docs/plans/` entries, but it should not become the place where new work starts. New work belongs in `prototypes/features/<slug>/`.
