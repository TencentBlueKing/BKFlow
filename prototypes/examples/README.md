# examples

## Purpose

`prototypes/examples/` is a small reference shelf for the component showcase. It is not for delivery.

## Contract

- `prototypes/examples/` is for reference only and is not for delivery.
- Keep component demos in `component-showcase.html`.
- Keep only `README.md` and `component-showcase.html` here.
- Do not put feature pages or masters here.
- `docs/specs/` and `docs/plans/` should not depend on examples for implementation trace.

## What goes here

- The component showcase page.
- Material that helps explain the prototype toolkit itself.

## What should not go here

- Feature delivery pages.
- Long-lived masters.
- Content that belongs to a spec/plan-backed feature directory.

## Relationship to `docs/specs` and `docs/plans`

`docs/specs/` and `docs/plans/` should not depend on this directory for delivery. Examples can support understanding, but the implementation trace should live in `masters/` or `features/`, with `features/_legacy/` reserved for migrated history only.
