# config-page

## Purpose

`prototypes/masters/config-page/` is the reusable master for long configuration forms. It emphasizes a multi-section form, a code or editor block, a disabled save state in the baseline template, and a dirty state for unsaved changes.

## What it covers

- Long-running admin forms with several field groups.
- A code block or editor-style area for structured configuration.
- Disabled save behavior in the default template.
- Dirty-state treatment when the form has unsaved changes.
