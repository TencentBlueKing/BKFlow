# BKFlow 核心概念

## Overview

BKFlow 是面向平台的流程引擎，通过"空间"进行业务隔离，以"流程模板"定义执行蓝图，以"任务实例"驱动实际执行。理解这些核心概念是所有开发工作的基础。

## 资源层级

```
空间 (Space)
├── 流程模板 (Template) [有 source 标识多版本]
│   ├── 节点 (Node)
│   │   ├── 流程控制节点（开始/结束/网关）
│   │   └── 任务节点（绑定插件）
│   └── 全局变量 (Constants)
├── 任务实例 (Task)
│   └── 基于某个流程模板创建，是真实的业务执行
├── 决策表 (Decision Table)
└── 凭证 (Credential)
```

## 关键概念定义

### 空间（Space）

- BKFlow 业务隔离的顶级单位，不同空间数据不互通
- 必须绑定接入系统的 `APP_CODE`
- **约束**：推荐每个接入系统只对应一个空间

### 流程模板（Template）

- 任务执行的蓝图，定义节点顺序和执行逻辑
- 支持版本概念，通过 `source` 字段标识同一来源的多个版本
- 同一 space 同一 source 下版本号不允许重复
- 单个流程可以有多个启用版本，支持基于指定版本创建任务

### 任务实例（Task）

- 基于流程模板创建的一次实际业务执行
- 可以被暂停、被强制终止
- 每个节点有明确的生命周期状态：未执行 → 正在执行 → 执行成功/执行失败

### 节点（Node）

**流程控制节点**（不包含执行逻辑，仅控制流向）：

- **开始节点**：标识流程开始
- **并行网关**：标识并行流程开始，必须与汇聚网关配对
- **分支网关（互斥网关）**：根据表达式动态选择唯一执行分支，必须与汇聚网关配对
- **条件并行网关**：满足分支条件的分支都会执行
- **汇聚网关**：标识并行或分支流程结束

> **关键约束**：
> - 并行网关和分支网关必须各自与汇聚网关一一配对，不允许交叉嵌套或不配对
> - 并行流程内可嵌入分支流程，分支流程内可嵌入并行流程，但汇聚网关必须严格对应

**任务节点**（包含实际执行逻辑）：

- 包含基础信息、输入参数、输出参数
- 输出参数可勾选生成全局变量
- 输入参数可引用全局变量

### 全局变量（Constants）

- 流程模板的公共参数，以 KEY 做唯一约束
- 可在任务节点输入参数和分支网关表达式中引用
- BKFlow 在任务执行时自动替换引用为实际值
- 变量值类型支持：`string`、`number`、`boolean`

### 插件（Plugin）

三种类型：

- **内置插件**：BKFlow 默认提供（暂停、定时、消息展示、审批、决策等）
- **API 插件**：接入系统开发者通过暴露特定协议接口自定义的插件，在 BKFlow 自动生成对应插件
- **蓝鲸插件**：对接蓝鲸插件服务，开发者通过蓝鲸开发者中心实现

## 三大核心功能

1. **流程编排和画布嵌入**：接入系统可嵌入 BKFlow 画布，支持自定义权限控制
2. **流程任务执行能力**：通过 API 实现流程任务的创建、执行和控制
3. **决策引擎能力**：在流程中集成规则管理和 DMN 决策

## 接入场景类型

集成方式：Web 服务集成（嵌入画布）或 SDK 集成（直接调用 API）
