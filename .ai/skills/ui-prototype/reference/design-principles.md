# 通用 UI 设计原则

> 来源：开源社区 [aussiegingersnap/cursor-skills](https://github.com/aussiegingersnap/cursor-skills)，
> 适用于需要更高设计标准时参考。日常原型迭代不强制要求。

## 设计方向选择

在编写代码前，确定界面的设计方向：

### 产品性格

- **精密与密度**（Precision & Density）— 紧凑间距、信息优先。适合高频使用的工具型产品。参考：Linear、Raycast
- **温暖与亲和**（Warmth & Approachability）— 充裕间距、柔和阴影。适合协作型产品。参考：Notion、Coda
- **专业与信任**（Sophistication & Trust）— 冷色调、层次深度。适合涉及敏感数据的产品。参考：Stripe、Mercury
- **功能与实用**（Utility & Function）— 柔和色板、功能密度。适合工具型产品。参考：GitHub、开发者工具

### 色彩基调

- **暖基调**（米色、暖灰）— 亲近感
- **冷基调**（石板色、蓝灰）— 专业感
- **纯中性**（纯灰、黑白）— 极简技术感
- **强调色** — 选择 ONE 个有意义的：蓝色=信任，绿色=成功，橙色=活力

## 工艺原则（质量底线）

### 4px 间距网格

所有间距基于 4px 基准：

| 值 | 用途 |
|----|------|
| 4px | 微间距（图标间隙） |
| 8px | 紧凑间距（组件内部） |
| 12px | 标准间距（相关元素之间） |
| 16px | 舒适间距（区块内边距） |
| 24px | 宽松间距（区块之间） |
| 32px | 大分隔 |

### 对称内边距

上下左右保持一致。例外：当水平方向确实需要更多空间时。

```css
/* 好 */
padding: 16px;
padding: 12px 16px;

/* 坏 */
padding: 24px 16px 12px 16px;
```

### 圆角一致性

选择一套系统并坚持使用：
- 锐利：4px, 6px, 8px
- 柔和：8px, 12px
- 极简：2px, 4px, 6px

### 深度与阴影策略

选择 ONE 种方式并统一使用：

- **纯边框（扁平）** — 干净、技术感、密度高
- **单层阴影** — 轻微浮起，不复杂
- **背景色区分** — 用不同背景色层级建立层次，不用阴影

### 排版层级

- 标题：600 字重，紧凑字间距
- 正文：400-500 字重，标准间距
- 标签：500 字重
- 字号梯度：12px, 13px, 14px(基准), 16px, 18px, 24px

### 数据用等宽字体

数字、ID、代码、时间戳使用等宽字体，配合 `tabular-nums` 确保列对齐。

### 动画

- 微交互 150ms，较大过渡 200-250ms
- 缓动函数：`cubic-bezier(0.25, 1, 0.5, 1)`
- 企业级 UI 不使用弹跳效果

### 颜色只用于传达意义

灰色构建结构。颜色仅在传达信息时出现：状态、操作、错误、成功。装饰性颜色是噪音。

## 反模式

- 夸张的阴影（`box-shadow: 0 25px 50px...`）
- 小元素用大圆角（16px+）
- 无理由的非对称内边距
- 装饰性渐变
- 同一界面使用多个强调色
- 区块间距过大（> 48px）
